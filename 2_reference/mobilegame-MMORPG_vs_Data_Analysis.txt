1. 연구 주제
모바일 MMORPG에서 조작 복잡도는 체감 난이도를 어떻게 재구성하는가
1-1. 로스트아크 모바일과 아이온2 비교 분석

2. 연구 배경
모바일 MMORPG는 동일 장르임에도 체감 난이도가 크게 다르게 나타난다
2-1. 기존 인식은 난이도가 콘텐츠에 의해 결정된다는 것이다
2-2. 하지만 모바일 환경에서는 입력 방식과 시스템이 플레이 경험에 영향을 준다
2-3. 따라서 난이도는 조작 시스템에 의해 왜곡될 가능성이 존재한다

3. 핵심 가설
체감 난이도는 콘텐츠가 아니라 조작 복잡도에 의해 결정된다
3-1. CCS가 증가할수록 피격률이 증가한다
3-2. CCS가 일정 수준을 넘으면 유저는 회피를 포기한다
3-3. 자동화 시스템은 체감 난이도를 실제보다 낮춘다

4. CCS 정의
CCS는 Control Complexity Score로, 조작 복잡도를 수치화한 지표이다
4-1. 플레이어가 요구받는 입력 부담을 정량화한다
4-2. 단순 버튼 수가 아니라 조작 난이도를 포함한다

5. CCS 구성 요소
5-1. 입력 다양성
5-1-1. 스킬 버튼 수
5-1-2. 입력 방식 종류

5-2. 동시 입력 요구
5-2-1. 이동, 스킬, 회피, 카메라 동시 조작 여부

5-3. 반응 시간 요구
5-3-1. 패턴 대응에 필요한 시간

5-4. 조준 복잡도
5-4-1. 자동 타겟 여부
5-4-2. 방향 지정 필요성

5-5. 카메라 조작 부담
5-5-1. 시점 제어 필요 여부
5-5-2. 조작 빈도

6. CCS 수식
CCS는 다음 요소들의 가중 합으로 계산된다

6-1. 입력 다양성
6-2. 동시 입력
6-3. 반응 시간
6-4. 조준 복잡도
6-5. 카메라 조작
6-6. 각 요소는 가중치를 가지며 실험을 통해 조정된다

7. 데이터 수집 방법
본 연구는 정식 서비스 데이터가 아닌 관측 데이터를 사용한다
7-1. 영상 데이터 기준

7-1-1. 실제 플레이 장면이 포함된 영상만 사용한다
7-1-2. 조작 과정이 명확히 드러나는 영상만 포함한다
7-1-3. 피격, 회피, 실패가 관측 가능한 영상만 사용한다
7-1-4. 다음 영상은 제외한다
홍보 영상
트레일러
편집된 하이라이트

7-2. 게임별 데이터 기준
7-2-1. 로스트아크 모바일

7-2-1-1. CBT 플레이 영상만 사용한다

7-2-2. 아이온2

7-2-2-1. 출시일 기준 전후 2주 데이터만 사용한다
7-2-2-2. 초기 플레이 영상만 분석한다
7-2-2-3. 유저 숙련도와 메타 영향을 최소화하기 위함이다

7-3. 커뮤니티 데이터 수집 기준

7-3-1. 커뮤니티 데이터는 조작 체감과 난이도 인식 분석을 위해 사용한다

7-3-2. 데이터 출처는 다음과 같다

DCInside
Inven
Reddit
YouTube 댓글

7-4. 커뮤니티 데이터 필터링 기준

7-4-1. 조작 관련 표현 포함 데이터만 사용한다
예: 조작이 어렵다, 손이 바쁘다, 컨트롤이 힘들다

7-4-2. 난이도 관련 표현 포함 데이터만 사용한다
예: 어렵다, 쉽다, 빡세다

7-4-3. 자동화 관련 표현 포함 데이터만 사용한다
예: 자동이 더 편하다, 수동이 힘들다

7-5. Reddit 데이터 처리 기준

7-5-1. Reddit 데이터는 보조 데이터로 활용한다
7-5-2. 전체 번역이 아닌 키워드 기반 분석을 수행한다

7-5-3. 주요 키워드

difficulty, hard, easy
control, input
auto, manual

7-5-4. 자동 번역은 보조적으로 사용하며, 의미 해석보다 빈도 분석에 집중한다.

8. 검증 방법

8-1. CCS 산출
8-1-1. 영상 기반 조작 요소 추출
8-1-2. 요소별 점수화

8-2. 난이도 지표 정의
8-2-1. 피격률
8-2-2. 회피 성공률
8-2-3. 사망까지 시간
8-2-4. 스킬 실패 빈도

8-3. 행동 변화 분석
8-3-1. 회피 시도 감소 여부
8-3-2. 자동 플레이 의존도
8-3-3. 포지셔닝 단순화

8-4. 상관관계 분석
8-4-1. CCS와 피격률
8-4-2. CCS와 회피 성공률
8-4-3. CCS와 플레이 방식

9. 난이도 왜곡 개념
난이도 왜곡은 실제 난이도와 체감 난이도의 차이를 의미한다

9-1. 과대평가
9-1-1. 콘텐츠는 쉬우나 조작이 어려운 경우

9-2. 과소평가
9-2-1. 콘텐츠는 어려우나 자동화로 인해 쉽게 느껴지는 경우

10. 예상 인사이트

10-1. 난이도는 입력 방식에 의해 재구성된다
10-2. 유저는 난이도가 높아지면 전략을 단순화한다
10-3. 모바일 MMORPG는 구조적으로 난이도 왜곡이 발생한다

11. 예상 결과

11-1. CCS 증가 시 피격률 증가
11-2. CCS 증가 시 회피 시도 감소
11-3. 자동화 시스템은 CCS 영향 감소

12. 결론
모바일 MMORPG에서 난이도는 인터페이스의 산물이다

12-1. 난이도 설계는 UX 문제로 확장된다
12-2. 모바일 환경은 게임 구조를 변화시킨다
12-3. 자동화 시스템은 난이도를 재조정한다

13. 최종 요약
유저는 게임을 못하는 것이 아니라 조작 시스템 때문에 못하게 된다
